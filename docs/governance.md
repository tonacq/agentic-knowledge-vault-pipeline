# Source rights and responsible use

This project separates what was technically demonstrated in a private prototype from what should be distributed as a public tool.

## Public repository boundary

This repository may publish:

- original architecture and analysis;
- sanitised code and configuration;
- deployment and recovery guidance;
- manifests and QA patterns using synthetic data;
- a small authorised or openly licensed demonstration corpus; and
- short, attributed excerpts where legally appropriate.

It will not publish:

- a third-party creator's complete transcript catalogue;
- the complete derivative vault built from that catalogue;
- credentials, cookies or production configuration; or
- instructions presented as permission to ignore platform terms or source rights.

## Intended input model

The reusable version is intended for:

- material owned by the operator;
- source files supplied by the user;
- appropriately licensed material; or
- content used with the creator's authorisation.

Users remain responsible for confirming their rights to acquire, process and distribute source material and for complying with applicable platform terms and law.

This is a project design boundary, not legal advice.
